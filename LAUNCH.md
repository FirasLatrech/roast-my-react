## Show HN

**Title:** Show HN: roast-my-react – a CLI that audits your React app and roasts you for it

**Body:**

> I kept ignoring my Lighthouse reports because they're a wall of JSON and zero fun to read. So I built a CLI that collects the *real* data — Lighthouse scores, total JS shipped, the largest chunks, excessive React re-renders (via a bippy-style `__REACT_DEVTOOLS_GLOBAL_HOOK__`), and axe-core accessibility violations — then hands it to an LLM and asks for one brutal one-liner per issue, each paired with a concrete fix.
>
> It runs against any local dev server (`npx roast-my-react --url http://localhost:3000`), needs zero config, and downloads its own headless Chromium on first run. The AI step uses a free model by default (Groq / Llama 3.3 70B), or any OpenAI-compatible endpoint including local Ollama — and without a key it still prints the raw findings.
>
> It's MIT, the whole thing is ~100KB, and there's an optional self-contained HTML report. Would love feedback on the audit accuracy and what other signals are worth surfacing (hydration time, Core Web Vitals field data, image weight).
