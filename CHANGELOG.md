# Changelog

All notable changes to **roast-my-react** are documented here. This project adheres to
[Semantic Versioning](https://semver.org) and the [Keep a Changelog](https://keepachangelog.com) format.

## [0.9.0]

### Added

- **Config file** — drop a `roast.config.json` (or a `"roast"` key in `package.json`) so teams don't
  repeat flags. CLI flags always override config; config overrides built-in defaults.
- Documented version history in this CHANGELOG.

### Changed

- **Cross-route dedup** — when auditing multiple `--routes`, an issue that appears on several routes is
  now reported once, annotated with how many routes it affects (instead of duplicating per route).

## [0.8.0]

### Added

- **Arrow-key server picker** — when several dev servers are running, choose with ↑/↓ (or `j`/`k`),
  number keys to jump, and Enter to confirm. Zero-dependency raw-mode TUI; falls back gracefully on
  non-interactive stdin.

## [0.7.0]

### Changed

- **Terminal redesign** inspired by react-doctor / clig.dev: a boxed **Diagnosis** summary panel,
  issues grouped by category with counts, calmer color (honors `NO_COLOR`), and a closing
  prescription line.
- `--help` now leads with an **Examples** section.

## [0.6.0]

### Added

- The interactive picker now lists **every** running server (not just multiple config apps).
- **Lighthouse retry** — one automatic retry smooths over cold-start flakes.

### Changed

- The shareable PNG **card is generated by default** (use `--no-card` to skip).
- The footer now shows the **card** path instead of a raw screenshot.

## [0.5.0]

### Added

- **Recursive monorepo detection** — reads `project.json` (Nx), `angular.json`, `package.json`
  scripts, and vite/next/webpack configs across the whole repo, mapping each app to its port + name.
- **Parallel multi-route auditing** with a bounded concurrency pool.

### Fixed

- **Lighthouse mutex** — Lighthouse uses process-global state and can't run concurrently; it's now
  serialized while the rest of each route's crawl still overlaps.

### Changed

- HTML report redesigned to match the card aesthetic (grade badge, scores, Core Web Vitals).

## [0.4.0]

### Added

- **Core Web Vitals** (LCP / CLS / TBT / FCP / SI / TTI) in the terminal and report.
- **Baseline / compare** (`--save-baseline`, `--baseline`) to track regressions.
- **Multi-route auditing** (`--routes`).
- **Shareable PNG card** (1200×630).
- **GitHub Action** template to comment a roast on PRs.
- **Config-based port detection** for monorepos (first version).

### Changed

- Default model is now configurable and tuned for the JSON roast contract.

## [0.3.0]

### Added

- **Auto-install Chromium** on first run — zero setup for `npx`.
- **Login support** (`roast-my-react login`) plus `--auth` / `--header` for dashboards behind auth.
- **`--fast`** to skip Lighthouse, and **`--fix`** to hand findings to a coding agent
  (Claude Code / Codex / opencode).

## [0.2.0]

### Added

- **Auto-detect** a running dev server across common framework ports when `--url` is omitted.

## [0.1.0]

### Added

- Initial release: Lighthouse + bundle-size + React re-render + axe-core audit, an AI roast with a
  fix per issue (free Groq model by default, any OpenAI-compatible API supported), polished terminal
  output, and a standalone HTML report.
